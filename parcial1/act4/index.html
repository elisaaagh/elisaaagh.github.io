Actividad 4

Mecanismos de defensa en red

Introducción

En esta práctica se implementan reglas de filtrado utilizando iptables con el objetivo de fortalecer la seguridad perimetral de la red. Se aplican políticas restrictivas y posteriormente se habilitan servicios necesarios como DNS, correo electrónico y HTTP.

Desarrollo técnico

1. Política restrictiva

iptables -P INPUT DROP
iptables -P OUTPUT DROP
iptables -P FORWARD DROP

2. Conexiones establecidas

iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

3. DNS saliente

iptables -A OUTPUT -p tcp --dport 53 -j ACCEPT

4. Correo entrante

iptables -A INPUT -p tcp -d 192.1.2.10 --dport 25 -j ACCEPT

5. Correo saliente

iptables -A OUTPUT -p tcp -s 192.1.2.10 --dport 25 -j ACCEPT

6. HTTP hacia servidor web

iptables -A INPUT -p tcp -d 192.1.2.11 --dport 80 -j ACCEPT

7. HTTP desde la red local

iptables -A INPUT -p tcp -s 192.1.2.0/24 --dport 80 -j ACCEPT

Opinión / reflexión técnica

La aplicación de listas de control de acceso demuestra la importancia de trabajar bajo el principio de mínimo privilegio. Definir primero una política de denegación total y después habilitar únicamente los servicios necesarios reduce significativamente la superficie de ataque y permite un mejor control del tráfico.
